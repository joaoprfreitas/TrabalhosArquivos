------ ARQUIVO TIPO 1 ------

**** Registro de cabeçalho:
- Status
- Topo
- Descricao *
- Proximo RRN
- Numero Registros Removidos

Registro de cabeçalho deve seguir estritamente a ordem acima
Os campos são de tamanho fixo (Valores armazenados não devem terminar em \0)

**** Registro de Dados (tamanho fixo) - 97 bytes:
-> Campos de tamanho fixo e variável
   - para tamanho variável, utilizar o método indicador de tamanho

Tamanho fixo:
- id (código identificador)
- ano
- quantidade de veiculos
- sigla
Tamanho variável:
- cidade
- marca
- modelo

Os  seguintes campos são necessários para o gerenciamento do registro de dados:
- removido
- proximo RRN
- tamanho cidade
- tamanho marca
- tamanho modelo

------ ARQUIVO TIPO 2 ------
**** Registro de cabeçalho (190 bytes):
- Status
- Topo
- Descricao
- Proximo Byte Offset
- Numero de registros removidos

Registro de cabeçalho deve seguir estritamente a ordem acima
Os campos são de tamanho fixo (Valores armazenados não devem terminar em \0)

**** Registro de dados ():
- id
- ano
- quantidade de veiculos
- sigla

tamanho variável:
- cidade
- marca
- modelo

Os  seguintes campos são necessários para o gerenciamento do registro de dados:
- removido
- tamanho do registro
- byte offset do proximo registro logicamente removido
- tamanho cidade
- tamanho marca
- tamanho modelo
